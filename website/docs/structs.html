Custom Types

Midori uses the type keyword to define both structs (product types) and enums (sum types). This gives you a unified way to model your data.

Structs with type

Define a struct with named fields:

type User {
    name: Str,
    email: Str,
    age: Int,
}

// Creating an instance
let user = User {
    name: "Alice",
    email: "alice@example.com",
    age: 30,
}

// Accessing fields
println(user.name)   // "Alice"
println(user.age)    30

Enums with type

Define an enum with variants that can carry data:

type Status {
    Active,
    Inactive,
    Banned(reason: Str),
}

let status = Active
let banned = Status.Banned("spam")

Methods with impl

Add methods to any type using an impl block. The first parameter is always self:

type Vec2 {
    x: Float,
    y: Float,
}

impl Vec2 {
    // Constructor
    fn new(x: Float, y: Float) -> Vec2 {
        Vec2 { x, y }  // shorthand field init
    }

    // Method taking &self (immutable)
    fn magnitude(self) -> Float {
        (self.x * self.x + self.y * self.y).sqrt()
    }

    // Method taking &mut self
    fn normalize(mut self) {
        let mag = self.magnitude()
        self.x = self.x / mag
        self.y = self.y / mag
    }
}

let v = Vec2.new(3.0, 4.0)
println(v.magnitude())  // 5.0

Traits

Traits define shared behavior across types. Implement a trait for any type with impl:

// Define a trait
trait Describable {
    fn describe(self) -> Str
}

// Implement the trait for a type
impl Describable for User {
    fn describe(self) -> Str {
        "User {self.name}, age {self.age}"
    }
}

impl Describable for Vec2 {
    fn describe(self) -> Str {
        "Vec2({self.x}, {self.y})"
    }
}

// Use the trait
fn print_description(item: impl Describable) {
    println(item.describe())
}

print_description(user)  // "User Alice, age 30"
print_description(v)     // "Vec2(3.0, 4.0)"

Pattern Matching on Enums

Enums shine with pattern matching. The compiler ensures all variants are handled exhaustively:

type Tree[T] {
    Leaf(value: T),
    Node(left: Box[Tree[T]], right: Box[Tree[T]]),
}

fn depth[T](tree: Tree[T]) -> Int {
    match tree {
        Leaf(_) => 1,
        Node(left, right) => {
            1 + max(depth(left), depth(right))
        },
    }
}

// Enum with destructuring
type Message {
    Quit,
    Move(x: Int, y: Int),
    Write(text: Str),
    ChangeColor(r: Int, g: Int, b: Int),
}

fn handle(msg: Message) {
    match msg {
        Quit => println("goodbye"),
        Move(x, y) => println("move to ({x}, {y})"),
        Write(text) => println("write: {text}"),
        ChangeColor(r, g, b) => {
            println("color: rgb({r}, {g}, {b})")
        },
    }
}

Generic Types

Types can be parameterized with generics:

type Pair[A, B] {
    first: A,
    second: B,
}

impl[A, B] Pair[A, B] {
    fn swap(self) -> Pair[B, A] {
        Pair { first: self.second, second: self.first }
    }
}

let pair = Pair { first: "hello", second: 42 }
let swapped = pair.swap()
println(swapped.first)  // 42
println(swapped.second) // "hello"
