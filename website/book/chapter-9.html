Chapter 9: Pattern Matching

Match Expressions

Pattern matching is one of Midori's most powerful features. The match expression compares a value against a series of patterns, executing the first matching arm. The compiler checks that all cases are covered — no more runtime "undefined" surprises.

match number {
    0 => "none",
    1 => "one",
    2 => "a couple",
    _  => "many",  // wildcard — matches anything
}

Destructuring

Match arms can destructure complex data types, binding parts of the value to variables. This works for structs, enums, tuples, and lists. Destructuring also works in let bindings and function parameters.

struct Point { x: Int, y: Int }

fn describe(p: Point) -> Str {
    match p {
        Point { x: 0, y: 0 } => "origin",
        Point { x: 0, y }     => "on y-axis at {y}",
        Point { x, y: 0 }     => "on x-axis at {x}",
        Point { x, y }           => "at ({x}, {y})",
    }
}

// Destructuring in let bindings
let Point { x, y } = origin

Guards

Match arms can have guards — additional conditions written after if that must be true for the arm to match. This lets you express complex conditional logic concisely.

fn classify(n: Int) -> Str {
    match n {
        0 => "zero",
        n if n < 0 => "negative",
        n if is_perfect_square(n) => "perfect square",
        _ => "positive",
    }
}

Matching on Enums

Pattern matching on enums is where the feature truly shines. The compiler ensures every variant is handled, and each arm can extract the associated data.

enum Status {
    Active,
    Inactive(Str),
    Banned { reason: Str, until: Str },
}

fn render(status: Status) -> Str {
    match status {
        Active                    => "Account is active",
        Inactive(reason)          => "Inactive: {reason}",
        Banned { reason, until }   => "Banned until {until}: {reason}",
    }
}

Combining Patterns

The pipe operator | inside a pattern matches multiple alternatives. This is useful when several variants should produce the same result.

fn is_weekend(day: Weekday) -> Bool {
    match day {
        Saturday | Sunday => true,
        _ => false,
    }
}

// Nested pattern matching
match data {
    Some(Ok(value)) => process(value),
    Some(Err(e))    => log_error(e),
    None             => "no data",
}
