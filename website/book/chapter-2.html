Chapter 2: Getting Started

Installation

The Midori compiler is distributed as a single binary. You can download the latest release from the GitHub releases page, or build from source if you prefer. On macOS you can install via Homebrew:

# Install Midori on macOS
brew install midori-lang/midori

On Linux, download the tarball and add the midori binary to your PATH. Windows users can use the installer from the releases page. After installation, verify everything works:

# Check the compiler version
midori --version

Hello, World!

Create a new file called hello.mi and open it in your editor. Midori source files use the .mi extension. Type the following:

fn main() {
    println("Hello, World!")
}

Every Midori program starts with a main function. The println function writes text to standard output. Notice there are no semicolons — Midori uses newlines for statement separation, keeping the code clean and minimal.

To compile and run your program, use the run subcommand:

midori run hello.mi

You should see Hello, World! printed to your terminal. Congratulations — you have written your first Midori program!

The Compiler Toolchain

The midori command provides several subcommands:

  • midori build — Compile the project to a binary.
  • midori run — Compile and run in one step.
  • midori test — Run tests for the current project.
  • midori fmt — Format source code according to standard style.
  • midori new — Scaffold a new project with the standard directory layout.

Let's create a proper project using midori new:

# Create a new project called "my-project"
midori new my-project
cd my-project

This creates the following structure:

my-project/
  src/
    main.mi
  midori.toml

The midori.toml file is the project manifest, similar to Cargo.toml in Rust or package.json in Node.js. It contains the project name, version, dependencies, and compiler settings.

# my-project/src/main.mi
fn main() {
    let name = "Midori"
    println("Welcome to {name}!" + " 🍃")
}

Build and run the project with:

midori run
