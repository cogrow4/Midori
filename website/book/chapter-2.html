Chapter 2: Getting Started

Installation

Midori is built from source. You need Rust and a C compiler (Clang or GCC) installed on your system.

# Clone the repository
git clone https://github.com/cogrow4/Midori.git
cd Midori

# Build and install
./install.sh
# Installs to ~/.local/bin/midori

The install.sh script builds the compiler and copies it to your PATH. You can also use cargo build --release in the compiler/ directory if you prefer to manage the binary yourself.

After installation, verify everything works:

# Check the compiler version
midori --version

Hello, World!

Create a new file called hello.mi and open it in your editor. Midori source files use the .mi extension. Type the following:

fn main() {
    println("Hello, World!")
}

Now run it with the Midori compiler:

$ midori hello.mi
Hello, World!

The midori command compiles the source file to C, links it with the runtime library, and executes the resulting binary. You can also build a standalone binary:

$ midori build hello.mi
$ ./hello
Hello, World!

Creating a Project

For larger programs, you can create a project scaffold:

$ midori new my-project
$ cd my-project
$ midori run main.mi

The midori new command creates a directory with a main.mi file and a placeholder README.md.

Watch Mode

While developing, use midori dev to automatically recompile and re-run whenever you save changes:

$ midori dev hello.mi
# File watcher polls every 500ms
# Re-runs on change
