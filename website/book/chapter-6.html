Chapter 6: Functions

Function Definitions

Functions are defined with the fn keyword. The function name, parameters, optional return type, and body form the complete definition. Midori uses -> to separate parameters from the return type.

fn greet(name: Str) -> Str {
    "Hello, {name}!"
}

fn main() {
    println(greet("World"))
}

Parameters and Return Types

Each parameter has a name followed by a colon and its type. If a function returns nothing, the return type and arrow can be omitted — it implicitly returns Unit. You can also define default parameter values.

fn power(base: Int, exp: Int = 2) -> Int {
    if exp == 0 { 1 }
    else { base * power(base, exp - 1) }
}

power(3)     // 9, uses default exp=2
power(3, 3)  // 27

Implicit Returns

Because blocks are expressions, the last expression in a function body is automatically its return value. There is no return keyword — simply let the final expression speak. For early exits, a bare expression works the same way.

fn factorial(n: Int) -> Int {
    if n <= 1 { 1 }
    else { n * factorial(n - 1) }
}

Closures

Closures — anonymous functions that capture their environment — are defined with the same fn syntax but without a name. They can be assigned to variables, passed as arguments, and returned from functions. Closures can capture variables by reference or by value.

let multiplier = fn(x: Int) { x * 2 }
println(multiplier(5))  // 10

// Capturing environment
let offset = 10
let add_offset = fn(x: Int) { x + offset }
println(add_offset(3))  // 13

Pipe Operator

The pipe operator |> passes a value as the argument to a function, making chains of transformations read naturally from left to right. This is especially useful for data processing pipelines.

let result = "hello world"
    |> uppercase
    |> split(" ")
    |> first

// Equivalent to: first(split(uppercase("hello world"), " "))

let nums = [1, 2, 3, 4, 5]
let sum_of_squares = nums
    |> filter(fn(x) { x % 2 == 0 })
    |> map(fn(x) { x * x })
    |> sum
