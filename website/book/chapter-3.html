Chapter 3: Basic Syntax

Comments

Midori supports two kinds of comments. Line comments start with // and extend to the end of the line. Block comments are enclosed between /* and */ and can span multiple lines. Documentation comments use three slashes /// and are used by the documentation generator.

// This is a line comment

/* This is a block comment
   that spans multiple lines */

/// Documentation comment for the function below
fn greet() { }

Literals

Midori provides literals for all primitive types. Integers can be written in decimal, hexadecimal (0x), octal (0o), or binary (0b). Floats support decimal and scientific notation. Strings are enclosed in double quotes and support interpolation with curly braces.

42          // Int literal (decimal)
0xFF        // Int literal (hexadecimal)
0b1010      // Int literal (binary)
3.14        // Float literal
1.5e10      // Float literal (scientific notation)
"hello"     // String literal
"val = {x}" // String with interpolation
true        // Bool literal

Operators

Midori supports the usual arithmetic, comparison, and logical operators. Arithmetic operators include +, -, *, /, and % (modulo). Comparison operators produce a Bool and include ==, !=, <, >, <=, and >=.

Logical operators are && (and), || (or), and ! (not). Midori also has a pipe operator |> for chaining function calls, which we will cover in Chapter 6.

let sum        = 10 + 5    // 15
let divisible  = 10 % 3    // 1
let is_equal   = "a" == "b"  // false
let both_true  = true && false  // false

Expressions and Statements

One of Midori's most important design choices: almost everything is an expression. Blocks delimited by braces are expressions that return the value of their last line. This means if, match, and even bare blocks all produce values.

let message = if age >= 18 {
    "adult"
} else {
    "minor"
}

// Blocks as expressions
let result = {
    let x = 5
    x * 2
}  // result is 10

Statements are expressions whose result is discarded. Variable bindings with let, function definitions with fn, and expression statements (an expression followed by a newline) are all statements. Midori does not use semicolons — newlines serve as separators in most cases.

// This is idiomatic Midori — expression-oriented and clean
fn classify(score: Int) -> Str {
    if score >= 90 { "A" }
    elif score >= 80 { "B" }
    elif score >= 70 { "C" }
    else              { "D" }
}
