Chapter 5: Types

Primitive Types

Midori provides a set of built-in primitive types that form the foundation of all programs. These types are deeply integrated into the compiler and have special syntax for their literals.

  • Int — Signed integer, platform-sized (64-bit on most systems).
  • Float — Double-precision IEEE 754 floating point.
  • Bool — Boolean values true and false.
  • Str — UTF-8 encoded string with interpolation support.
  • Char — A single Unicode code point.
  • Unit — The void type, used for functions that return nothing.
let age:    Int   = 25
let pi:     Float = 3.14159
let active: Bool  = true
let name:   Str   = "Midori"
let grade:  Char  = 'A'

Type Annotations

While Midori infers types in most cases, you can add explicit type annotations with a colon after the variable name. Annotations are useful for documentation, enforcing interfaces, or disambiguating when inference is ambiguous.

fn add(a: Int, b: Int) -> Int {
    a + b
}

let result: Int = add(3, 4)

Type Inference in Action

The type inference engine works across function boundaries and through complex expressions. It uses Hindley-Milner-style inference, the same algorithm that powers languages like Haskell and OCaml. This means you almost never need to write types, yet every expression is fully checked at compile time.

// No type annotations needed — everything is inferred
fn square(x) { x * x }

fn main() {
    let n = square(5)     // n is Int
    let f = square(2.5)   // f is Float
    println(n + f)
}

Custom Types

Beyond primitives, Midori lets you define your own types using struct and enum. These are covered in depth in Chapter 8, but here is a quick preview:

struct User {
    name: Str,
    age:  Int,
}

enum Color {
    Red,
    Green,
    Blue,
    Rgb(Int, Int, Int),
}

Type Aliases

You can create type aliases with the type keyword to give descriptive names to existing types or simplify complex generic signatures.

type UserId = Int
type Callback<T> = fn(T) -> Unit

fn lookup(id: UserId) -> Option<User> {
    // ...
}
