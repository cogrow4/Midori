Chapter 11: Traits and Generics

Traits

Traits are Midori's mechanism for defining shared behavior across types. A trait declares a set of method signatures that implementing types must provide. This is similar to interfaces in other languages, but with more power — traits can have default implementations, associated types, and supertraits.

trait Describable {
    fn describe(self) -> Str
}

impl Describable for User {
    fn describe(self) -> Str {
        "User: {self.name} (age {self.age})"
    }
}

Default Implementations

Traits can provide default implementations for some methods. Types that implement the trait can override these or use the defaults. This is useful for providing sensible defaults while still allowing customization.

trait Greeter {
    fn greeting(self) -> Str { "Hello!" }
    fn greet(self) {
        println(self.greeting())
    }
}

impl Greeter for User {
    // Uses default greet, overrides greeting
    fn greeting(self) -> Str {
        "Hi, I'm {self.name}!"
    }
}

Generic Functions

Generics allow functions and types to work with any type while maintaining full type safety. Type parameters are declared in angle brackets <T> and can be constrained by traits. Midori's generics are monomorphized — each concrete instantiation generates specialized code with no runtime overhead.

// Identity function — works for any type
fn id<T>(value: T) -> T { value }

// Generic with trait bound
fn print_descriptions<T: Describable>(items: [T]) {
    for item in items {
        println(item.describe())
    }
}

Generic Structs and Enums

Types themselves can be generic. This is how Option<T>, Result<T, E>, and collection types like List<T> are defined in the standard library.

struct Pair<A, B> {
    first: A,
    second: B,
}

enum Result<T, E> {
    Ok(T),
    Err(E),
}

impl<A, B> Pair<A, B> {
    fn swap(self) -> Pair<B, A> {
        Pair { first: self.second, second: self.first }
    }
}

Supertraits

Traits can extend other traits, forming a hierarchy. A type implementing a subtrait must also implement all supertraits.

trait Printable {
    fn print(self)
}

trait JsonSerializable: Printable {
    fn to_json(self) -> Str
}

impl JsonSerializable for Config {
    fn print(self)   { println("{self}") }
    fn to_json(self) -> Str {
        "{ \"key\": \"{self.value}\" }"
    }
}
