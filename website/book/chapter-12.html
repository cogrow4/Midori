Chapter 12: Modules and Organization

The Module System

Midori's module system helps you organize code into logical units. Each file is a module, and directories create module hierarchies. The module path mirrors the file system path, making it easy to navigate large projects.

// src/math/basic.mi
fn add(a: Int, b: Int) -> Int { a + b }
fn subtract(a: Int, b: Int) -> Int { a - b }

Importing

Use the use keyword to bring items from other modules into scope. You can import specific names, use wildcards, or alias imports. The pub keyword controls visibility — items are private by default.

// src/main.mi
use math::basic::{add, subtract}
use math::advanced::*     // wildcard import
use io::File as FileHandle  // aliased import

fn main() {
    let sum = add(5, 3)
}

Visibility

By default, everything is private to its module. Use pub to expose items. Modules themselves need pub to be accessible from parent modules. Midori also supports pub(crate) for crate-level visibility.

// src/db/mi
pub fn connect() -> Connection { /* ... */ }

fn internal_validate() -> Bool { /* private */ }

pub struct Connection {
    pub host: Str,
    port: Int,  // field is private
}

Project Structure

A typical Midori project follows a standard layout. The midori.toml manifest defines metadata and dependencies. Source code lives in src/, tests in tests/, and examples in examples/.

my-project/
  midori.toml       # Project manifest
  src/
    main.mi         # Entry point
    lib.mi          # Library root (for library crate)
    math/
      basic.mi
      advanced.mi
  tests/
    integration.mi
  examples/
    demo.mi

The Module File

For a directory module (like math/), Midori looks for a mod.mi file inside. This file can re-export submodules or provide the module's public API.

// src/math/mod.mi
pub use basic::{add, subtract}
pub use advanced::{factorial, fibonacci}

// Now callers can do:
// use math::{add, factorial}

Dependencies

The midori.toml file manages external dependencies. Dependencies are fetched from the package registry and resolved at build time.

# midori.toml
[package]
name = "my-project"
version = "0.1.0"

[dependencies]
json = "2.1"
http = "0.5"
