Chapter 13: Building and Testing

Compilation

The midori build command compiles your project into a native binary. The compiler first translates Midori to C, then uses the system C compiler (Clang or GCC) to produce machine code. This two-phase approach gives Midori access to the entire C ecosystem while providing a modern language experience.

# Build in release mode with optimizations
midori build --release

# Build with debug symbols
midori build --debug

# Check for errors without generating code
midori check

Release mode enables all optimizations, resulting in faster but slower-to-compile binaries. Debug mode adds debug information and disables optimizations for a better debugging experience.

Running Tests

Midori has built-in test support. Test functions are annotated with the @test attribute and placed in test modules. The midori test command finds and runs all tests in the project.

// src/math/basic.mi
fn add(a: Int, b: Int) -> Int { a + b }

@test
fn test_add() {
    assert_eq(add(2, 3), 5)
    assert_eq(add(-1, 1), 0)
    assert_eq(add(0, 0), 0)
}

Test Organization

Tests can be placed alongside code using a tests module, or in a separate tests/ directory for integration tests. Unit tests go in the same file as the code they test; integration tests go in the tests/ directory and can only access the public API.

// Inline unit tests — tested with `midori test`
fn parse(input: Str) -> Option<Int> {
    // implementation
}

@test
fn test_parse_valid() {
    assert_eq(parse("42"), Some(42))
}

// tests/integration.mi — integration tests
use my_project

@test
fn end_to_end() {
    // test the public API
}

Test Assertions

Midori provides a rich set of assertion macros for testing. Beyond simple equality, you can check for expected errors, panics, and approximate floating-point values.

@test
fn test_advanced() {
    assert_eq(result, expected)
    assert_ne(result, wrong)
    assert_true(condition)
    assert_false(condition)

    // Assert that an expression returns an error
    assert_err(divide(1, 0))

    // Floating-point approximate equality
    assert_approx_eq(0.1 + 0.2, 0.3)
}

Benchmarking

Midori includes a built-in benchmarking framework. Use the @bench attribute to mark benchmark functions. The framework runs the function multiple times and reports timing statistics.

@bench
fn bench_sort() {
    let mut data = [3, 1, 4, 1, 5, 9, 2, 6]
    sort(&mut data)
}

# Run benchmarks
midori bench
