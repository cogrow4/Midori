Chapter 1: Introduction

What is Midori?

Midori is a compiled programming language designed from the ground up for developer joy. It combines the expressiveness of modern functional languages with the performance of systems languages, all wrapped in a syntax that feels familiar yet fresh. The name Midori — Japanese for "green" — reflects the project's commitment to sustainability: sustainable code, sustainable tooling, and a sustainable open-source community.

Midori compiles through C to native machine code, which means it runs everywhere C runs. The compiler is self-hosting — written in Midori itself — and produces efficient binaries without requiring a heavy runtime.

Design Philosophy

Every feature in Midori is guided by a few core principles:

  • Beauty matters. Code is read far more often than it is written. Clean, consistent syntax reduces cognitive load and makes maintenance a joy.
  • Safety without ceremony. Strong static types, immutability by default, and algebraic error handling eliminate entire categories of bugs without making you fight the type system.
  • Zero-cost abstractions. High-level features like closures, traits, and pattern matching compile down to the same efficient machine code you would write by hand.
  • Developer experience first. Clear error messages, fast compilation, and a batteries-included standard library keep you in flow state.
// A glimpse of Midori — this compiles and runs
fn greet(name: Str) -> Str {
    "Hello, {name}!"
}

fn main() {
    println(greet("Midori"))
}

Who This Book Is For

This book is for programmers of all experience levels who want to learn Midori. If you have experience with Rust, Swift, Kotlin, TypeScript, or Haskell, you will find many familiar concepts. But even if Midori is your first compiled language, the book starts from fundamentals and builds up gradually.

Each chapter includes runnable code examples. We recommend following along by installing the Midori compiler (covered in Chapter 2) and experimenting as you read.

// Midori's type system catches errors at compile time
fn divide(a: Int, b: Int) -> Result<Int, Str> {
    if b == 0 {
        Err("division by zero")
    } else {
        Ok(a / b)
    }
}

What You Will Learn

By the end of this book you will understand Midiro's entire type system, its pattern matching capabilities, its trait and generics system, and how to build real applications. You will also learn about the build system, testing, and the standard library so you can be productive from day one.

// Pattern matching on enums — a common Midori pattern
match divide(10, 2) {
    Ok(val)  => println("Result: {val}"),
    Err(msg) => println("Error: {msg}"),
}
